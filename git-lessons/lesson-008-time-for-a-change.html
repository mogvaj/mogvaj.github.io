DCN: How to Git

Lesson 8 - Time for a change!

  • Note 1: The following instructions are specific to the Green-side gitlab. These instructions also apply to the Red-side gitlab, however the URL is different, so keep that in mind.
  • Login to your UNET workstation (or Remote UNET, or UVDI, etc.)
  • Start up File Manager and navigate to your development folder (i.e., I called mine "development" under my "Documents" folder)
  • Navigate into the "hello-git" folder
  • Right click somewhere in your development folder and choose "Git bash here"
  • Check to see which branch you have activated:
      • $ (addNiceGreeting) git status On branch addNiceGreeting nothing to commit, working tree clean $ (addNiceGreeting)
  • If you aren't already on the "addNiceGreeting" branch, then check it out:
      • $ (master) git checkout addNiceGreeting Switched to branch 'addNiceGreeting' $ (addNiceGreeting)
  • Note: Any text editor can be used at this point. For these lessons, I am going to use Windows' Notepad...just for simplicity.
  • In File Manager, open the sub-folder named "hello-git"
  • Some things to notice:
    • There is a weird-looking sub-folder here called ".git" --- Don't go in there!
        • (The ".git" folder is where the Git client does all of its magic. After you've got more experience with Git, you can explore that sub-folder. I never do. Everything I need to do can be done with standard Git commands. Your mileage may vary.)
    • There is no indication in File Manager of the currently checked-out branch
        • (For now, to make sure you've got the right branch checked out, use the "git status" command in the Git bash prompt.)
  • Make a change!
    1. Open the "hello-git.html" file with Notepad
    2. Add "This is my first edit" as a new line under the main title:
        3. <!doctype html>
<html>
    <head>
        <title>Hello, Git</title>
    </head>
    <body>
        <h1>Hello, Git</h1>
        This is my first edit
    </body>
</html>
    3. Save the file and exit Notepad
  • Go back to the Git bash window and enter "git status":
      • $ (addNiceGreeting) git status On branch addNiceGreeting Changes not staged for commit: (use "git add ..." to update what will be committed) (use "git restore ..." to discard changes in working directory) modified: hello-git.html no changes added to commit (use "git add" and/or "git commit -a") $ (addNiceGreeting)
  • Our next lesson is actually a break... We're havin' STORY TIME!
  • DONE!
