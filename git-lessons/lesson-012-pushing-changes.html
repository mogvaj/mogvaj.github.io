DCN: How to Git

Lesson 12 - Pushing changes to the origin

  • Note: The following instructions are specific to the Green-side gitlab. These instructions also apply to the Red-side gitlab, however the URL is different, so keep that in mind.
  • To recap where we are:
    • In Lesson 6, we cloned the "hello-git" repository
    • In Lesson 7, we made a new branch to do some work in: "addNiceGreeting"
    • In Lesson 8, we made a change to a file in our local copy of the "hello-git" repository
    • In Lesson 10, we "staged" our branch's files for the next step
    • In Lesson 11, we took a snapshot of our branch by using the "git commit" command
    • In THIS lesson, we will upload the change we made to the "origin"...gitlab...

  • If Git Bash isn't running, yet, start it up and navigate to the directory where "hello-git.html" resides
      • If you don't have that directory, back up to Lesson 6 and make your way back here...see you in a few...
  • In Git Bash, do a "git status" to get your bearings:
      • $ (addNiceGreeting) git status On branch addNiceGreeting Changes not staged for commit: (use "git add ..." to update what will be committed) (use "git restore ..." to discard changes in working directory) modified: hello-git.html no changes added to commit (use "git add" and/or "git commit -a") $ (addNiceGreeting)
  • At the end of the coding day...we should always send our changes to the "origin"...in our case, to gitlab
  • Use the "git push" command to send changes to the "origin":
      • $ (addNiceGreeting) git push origin addNiceGreeting Enumerating objects: 5, done. Counting objects: 100% (5/5), done.   • A couple things about git push origin addNiceGreeting:
    • git push - This tells your Git client to "upload any change commits"
    • origin - This tells your Git client that the place to upload to is the "origin", which was set to "gitlab" when we did the initial "clone"
    • addNiceGreeting - This tells your git client to also tell the origin that you've created a new branch called "addNiceGreeting"
