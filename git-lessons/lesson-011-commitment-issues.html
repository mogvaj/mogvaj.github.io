Lesson 11 - Commitment Issues

  • Note 1: The following instructions are specific to the Green-side gitlab. These instructions also apply to the Red-side gitlab, however the URL is different, so keep that in mind.
  • To recap where we are:
    • In Lesson 6, we cloned the "hello-git" repository
    • In Lesson 7, we made a new branch to do some work in: "addNiceGreeting"
    • In Lesson 8, we made a change to a file in our local copy of the "hello-git" repository
    • In Lesson 10, we "staged" our branch's files for the next step
    • In THIS lesson, we will commit our branch's current state

  • If Git Bash isn't running, yet, start it up and navigate to the directory where "hello-git.html" resides
      • If you don't have that directory, back up to Lesson 6 and make your way back here...see you in a few...
  • In Git Bash, do a "git status" to get your bearings:
      • $ (addNiceGreeting) git status On branch addNiceGreeting Changes to be committed: (use "git restore --staged ..." to unstage) modified: hello-git.html $ (addNiceGreeting)
  • Like we learned during Lesson 9: "Story Time", when we "commit", it's like taking a snapshot of the current state of your project that you staged with "git add -A" --- it's a place we can back up to later on (if we need to)
  • In Git Bash, take a snapshot by entering the "git commit" command:
      • $ (addNiceGreeting) git commit -m "Added a nice greeting to hello-git.html" [addNiceGreeting cab9669] Added a nice greeting to hello-git.html 1 file changed, 1 insertion(+) $ (addNiceGreeting)
  • A couple things about git commit -m "Added a nice greeting to hello-git.html":
    • git commit - This tells your Git client to "get out its camera to take a snapshot"
    • -m - "memo" or "message"
    • "Added a nice greeting to hello-git.html" - This is a short note to yourself describing this snapshot. If you ever do need to back up to a snapshot, you'll probably want to select the specific snapshot you want based on this memo.
