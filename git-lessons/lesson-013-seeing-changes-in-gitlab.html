DCN: How to Git

Lesson 13 - Seeing changes in Gitlab

  • Note: The following instructions are specific to the Green-side gitlab. These instructions also apply to the Red-side gitlab, however the URL is different, so keep that in mind.
  • To recap where we are:
    • In Lesson 6, we cloned the "hello-git" repository
    • In Lesson 7, we made a new branch to do some work in: "addNiceGreeting"
    • In Lesson 8, we made a change to a file in our local copy of the "hello-git" repository
    • In Lesson 10, we "staged" our branch's files for the next step
    • In Lesson 11, we took a snapshot of our branch by using the "git commit" command
    • In Lesson 12, we pushed changes to the origin (Gitlab)
    • In THIS lesson, we will look in Gitlab to see the changes we just pushed...

  • Login to your UNET workstation (or Remote UNET, or UVDI, etc.)
  • Start the Chrome browser and go to "https://gitlab.fbi.gov"
  • Login as necessary
  • In the upper left, click on "Projects"
  • In the dropdown menu that appears, click "Your projects"
  • A list of projects you have access to appears
  • Find the project named "Hello Git" and click on it
  • At the top of the page, you may see:
      • You pushed to addNiceGreeting at DCN / Hello Git 33 minutes ago
  • ...or something like that. That's our first clue that tells us we pushed successfully.
  • Under the project title, look at the section showing the number of branches belonging to this project:
      • 1 Commit - 2 Branches - 0 Tags - 72 KB Files - 72 KB Storage
  • It will show 2 branches. Those should be:
    • "master"
    • "addNiceGreeting"
  • Let's go ahead and check....
  • Click on the heading for "2 Branches"
  • You'll come to a page showing a listing of "Active Branches"
  • One of those is the "master" branch --- this is the "operational" branch --- the one presented to the public to use
  • The other should be the "addNiceGreeting" branch
  • So how do we get the changes we made put into the "master" branch so that they'll be available to the public?
  • There should be a button to the right of the "addNiceGreeting" entry labeled "Merge request"...
  • Don't click it yet! That's what we'll do in the next lesson
  • For now, you can see that the commit message you entered "Added a nice greeting to hello-git.html" is shown under the title of the "addNiceGreeting" branch
  • Click on the "addNiceGreeting" branch title
  • You'll be taken to the page for that branch
  • On this page, you can view the contents of the file by clicking on it
  • Next time, we'll make a request to merge the "addNiceGreeting" and "master" branches
  • DONE!
