DCN: How to Git

Lesson 7 - Let's Branch

  • Note 1: The following instructions are specific to the Green-side gitlab. These instructions also apply to the Red-side gitlab, however the URL is different, so keep that in mind.
  • Login to your UNET workstation (or Remote UNET, or UVDI, etc.)
  • Start up File Manager and navigate to your development folder (i.e., I called mine "development" under my "Documents" folder)
  • In lesson 6, you cloned the "Hello Git" project into this folder. There should be a folder named "hello-git"
  • Navigate into that "hello-git" folder
  • Right click somewhere in your development folder and choose "Git bash here"
  • Note:
    • Currently, we are in the "master" branch. Typically, we do NOT want to make changes to the "master" branch.
    • When making changes, we should do the following:
      1. Make a new branch
      2. Make desired changes in the new branch
      3. Push the changes to gitlab
      4. Request that the new branch be merged with the master branch
      5. Merge the branch into master
      6. Delete the no-longer-needed branch
  • Let's see what branches are available!
    • At the Git bash prompt, enter the following:
        • $ (master) git branch -a * master remotes/origin/HEAD -> origin/master remotes/origin/master $ (master)
  • Let's make a new branch to work on:
    • At the Git bash prompt, enter the following:
        • $ (master) git branch addNiceGreeting $ (master)
  • Check to see which branch you have activated:
      • $ (master) git status On branch master Your branch is up to date with 'origin/master'. nothing to commit, working tree clean $ (master)
  • Again, let's see what branches are available:
      • $ (master) git branch -a addNiceGreeting * master remotes/origin/HEAD -> origin/master remotes/origin/master $ (master)
  • Now, let's activate, or switch to, the branch we just created:
      • $ (master) git checkout addNiceGreeting Switched to branch 'addNiceGreeting' $ (addNiceGreeting)
  • Check to see which branch you have activated:
      • $ (addNiceGreeting) git status On branch addNiceGreeting nothing to commit, working tree clean $ (addNiceGreeting)
  • Next time, we'll make edits and, as the branch name suggests, add a nice greeting.
  • DONE!
