DCN: How to Git

Lesson 10 - Stand by for staging

  • Note 1: The following instructions are specific to the Green-side gitlab. These instructions also apply to the Red-side gitlab, however the URL is different, so keep that in mind.
  • At the end of Lesson 8, we had made and saved a change to "hello-git.html"
    • Story Time: arranging the collection

  • If Git Bash isn't running, yet, start it up and navigate to the directory where "hello-git.html" resides
  • Let's say you are at a stopping point (i.e., end of day, finished coding a specific portion, etc.)
  • The goal now is to send your changes to gitlab.fbi.gov
  • In Git Bash, stage your changes by adding them:
      • $ (addNiceGreeting) git add -A $ (addNiceGreeting)
  • Yeah...not much output on that one....
  • But your changes are now "Staged"
    • Story Time: Posing the collection in a certain arrangement
  • Do a "git status" to see where things are:
      • $ (addNiceGreeting) git status On branch addNiceGreeting Changes to be committed: (use "git restore --staged ..." to unstage) modified: hello-git.html $ (addNiceGreeting)
  • So, keep in your imagination: all the components of your local repo (on your hard drive) have been staged and someone is waiting with a camera to commit that arrangement to a photo...never to be forgotten...
  • DONE!
