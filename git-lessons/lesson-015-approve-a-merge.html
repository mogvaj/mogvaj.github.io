DCN: How to Git

Lesson 15 - Approve a merge

  • Note: The following instructions are specific to the Green-side gitlab. These instructions also apply to the Red-side gitlab, however the URL is different, so keep that in mind.
  • To recap where we are:
    • In Lesson 6, we cloned the "hello-git" repository
    • In Lesson 7, we made a new branch to do some work in: "addNiceGreeting"
    • In Lesson 8, we made a change to a file in our local copy of the "hello-git" repository
    • In Lesson 10, we "staged" our branch's files for the next step
    • In Lesson 11, we took a snapshot of our branch by using the "git commit" command
    • In Lesson 12, we pushed changes to the origin (Gitlab)
    • In Lesson 13, we checked to make sure the "addNiceGreeting" branch made it to Gitlab
    • In Lesson 14, we requested to merge the "addNiceGreeting" branch into the "master" branch
    • In THIS lesson, we will approve that request...

  • Login to your UNET workstation (or Remote UNET, or UVDI, etc.)
  • Start the Chrome browser and go to "https://gitlab.fbi.gov"
  • Login as necessary
  • In the upper left, click on "Projects"
  • In the dropdown menu that appears, click "Your projects"
  • A list of projects you have access to appears
  • Find the project named "Hello Git" and click on it
  • On the left side of the page, there should be an entry saying Merge requests      1
  • Click on that Merge requests entry on the left
  • You'll be shown a page listing the merge request you created in Lesson 14
  • Click on the title of the merge request --- it should be the last Commit message you entered:
      • Added a nice greeting to hello-git.html
  • You'll be taken to a page for that merge request
  • There are a few sections on the merge request page:
    • The first section gives you a link where you can view the changes that will happen as a result of the requested merge
    • The second section is for approvals. Those are optional in this case.
    • The third section has the button to execute the merge:
  • That third section is what we are interested in
  • About that third section:
    • The green checkmark in a circle means that the "addNiceGreeting" branch and the "master" branch have no conflicts --- in this case, "addNiceGreeting" will add one line of text which doesn't overlap anything already in the "master" branch
    • The button is what we'll click on to tell Gitlab to execute the merge
    • The ☑ Delete source branch tells Gitlab to delete the "addNiceGreeting" branch once the merge is complete
        • (We generally want to delete source branches --- branching should be done for specific additions or fixes, and once we are done with them, the branches shouldn't stick around. New additions or fixes should then be branched from the new version of "master".)
  • Go ahead and click on the button
  • The page will update to show that this merge record is completed
  • Next time, we'll sync your local repository to match the one now on Gitlab
  • DONE!
